🗻 Trans3D

Macro VBA pour MicroStation V8i — du plan 2D coté à la modélisation 3D terrain

Présentation Installation Formulaire Convertir Semer Points Key-in Touche F Dépannage

1. Présentation

Trans3D fournit trois commandes pour passer du plan 2D coté à la modélisation 3D dans MicroStation V8i. L'utilisateur travaille dans un fichier 3D actif avec un ou plusieurs fichiers 2D en référence (plans topo/VRD cotés).

CommandeKey-inDescription
Convertir vba run [Trans3D]Convertir Crée l'équivalent 3D d'un élément 2D d'une référence, avec altitudes interpolées sur l'abscisse curviligne. Supporte les altitudes intermédiaires (regards, changements de pente)
Semer vba run [Trans3D]Semer Matérialise des points cotés (cercle repère + texte altitude) le long d'un élément 3D existant, à distance fixe ou en N points également répartis
Points vba run [Trans3D]Points Crée un point 3D isolé depuis une position 2D (snap) et une altitude cliquée ou saisie

Principe général

Les trois commandes partagent un formulaire modeless unique (Tool Settings) et les mêmes règles de création :

  • Les sources d'altitude sont les textes simples, tags et cellules contenant une valeur numérique, dans le modèle actif et les références attachées
  • Le cercle repère est placé au Z réel (altitude calculée)
  • Le texte altitude est placé à Z=0.00 (lisible en vue top sans surcharger le modèle) — ou au Z réel si l'option est décochée
  • Le Z du point snappé n'est jamais utilisé : seule la cote fournit l'altitude

Types d'éléments acceptés (Convertir)

  • Ligne, ligne brisée, arc, courbe, B-spline, cercle, ellipse
  • Type conservé si possible : ligne → ligne 3D, ligne brisée → ligne brisée 3D (Z aux sommets), arc/cercle/ellipse à Z constant → type natif horizontal
  • Arc/courbe/B-spline avec pente → polyligne 3D discrétisée au pas paramétrable

2. Installation

2.0 Installation automatique (recommandée)

Télécharger le dépôt GitHub : téléchargement direct du ZIP (ou Code > Download ZIP sur la page du dépôt), décompresser, puis double-cliquer sur install.cmd : le projet Trans3D.mvba est copié dans le workspace avec chargement automatique au démarrage. Le Default.mvba personnel n'est jamais touché. Redémarrer MicroStation et c'est prêt. Les étapes 2.1 et 2.2 ci-dessous ne servent que pour une installation manuelle.

2.1 Ouvrir le projet VBA (installation manuelle)

  1. Dans MicroStation : Utilitaires > Macros > Gestionnaire de projets VBA
  2. Utilisez le projet Default (chargé automatiquement au démarrage) — ou créez un projet dédié si vous préférez
  3. Ouvrez l'éditeur VBA : Alt+F11

2.2 Importer les fichiers sources

  1. Dans l'éditeur VBA : Fichier > Importer un fichier (Ctrl+M)
  2. Importez, un par un, depuis le dossier src\ :
    FichierType attendu après import
    Trans3D.basModules
    RechercheAltitude.basModules
    RechercheElement.basModules
    CSettings.clsModules de classe
    CSymboTexte.clsModules de classe
    CSymboCercle.clsModules de classe
    CPointRef.clsModules de classe
    CCalcul.clsModules de classe
    CGraphique.clsModules de classe
    CParamSemis.clsModules de classe
    CAltitudeSelection.clsModules de classe
    CPlacerConvertir.clsModules de classe
    CPlacerSemis.clsModules de classe
    CPlacerPoints.clsModules de classe
    frmTrans3D.frm + .frxUserForm
  3. Débogage > Compiler : aucune erreur ne doit apparaître
  4. Enregistrez (Ctrl+S)
⚠️ Les fichiers .cls doivent arriver dans « Modules de classe ». S'ils arrivent dans « Modules » avec des lignes VERSION 1.0 CLASS ou Attribute VB_Name visibles, les fichiers sources ont des fins de ligne Unix (LF) : convertissez-les en fins de ligne Windows (CRLF) puis réimportez.
⚠️ Le fichier frmTrans3D.frx doit être dans le même dossier que frmTrans3D.frm lors de l'import. Sans lui, l'import échoue avec « Impossible de charger... ».

2.3 Chargement automatique

Avec le Default MVBA : rien à faire, il est chargé d'office par MicroStation à chaque démarrage.

Avec un .mvba dédié seulement : Utilities > Macros > Project Manager, clic droit sur le projet > Autoload.

2.4 Mise à jour

Pour récupérer une nouvelle version, la méthode la plus rapide est de rejouer install.cmd :

  1. Retélécharger le ZIP (ou git pull si vous avez cloné)
  2. Fermer MicroStation
  3. Double-cliquer sur install.cmd
  4. Relancer MicroStation

Le script remplace Trans3D.mvba dans le workspace : tous les modules, classes et formulaires sont mis à jour en un seul geste, sans risque d'oublier un fichier.

3. Formulaire (Tool Settings)

Au lancement de n'importe laquelle des trois commandes, le formulaire Trans3D s'ouvre et reste visible pendant toute la durée de la commande. Les paramètres sont modifiables à tout moment, même entre deux opérations. Le formulaire est commun aux trois commandes.

Texte altitude

  • Texte à Z=0.00 (case cochée par défaut) : le texte est créé à Z=0 (lisible en vue top). Case décochée : le texte est placé au Z réel de l'altitude
  • Dupliquer le style de la cote (case cochée par défaut) : police, taille, couleur, niveau et fond sont repris du texte source. Case décochée : les champs ci-dessous deviennent modifiables
  • Hauteur : hauteur du texte en unités maître
  • Coul. : index couleur MicroStation 0–255 (0=blanc, 1=bleu, 2=vert, 3=rouge, 4=jaune)
  • Niveau : liste des niveaux existants du fichier DGN
  • Police : nom de la police de caractères
  • Fond : activer/désactiver le fond (masque) du texte
  • Décimales : nombre de décimales (0–6, défaut 2)
  • Virgule : utiliser la virgule comme séparateur décimal (par défaut : point). La lecture des cotes 2D accepte les deux séparateurs

Cercle repère (au Z réel)

  • Créer le cercle : activer/désactiver la création du cercle repère
  • Diamètre : en unités maître (ex. 0.50)
  • Couleur : index MicroStation 0–255
  • Niveau : liste des niveaux existants du fichier DGN

Découpage (utilisé par Convertir)

  • Créer des points intermédiaires : si coché, des points cotés (cercle + texte) sont placés le long de l'élément 3D créé par Convertir
  • Distance fixe (m) : un point tous les N mètres le long de l'élément
  • Nombre de points : N points également répartis

Convertir

  • Pas discrétisation : pas de discrétisation pour les courbes avec pente (arc, B-spline → polyligne 3D). Plus le pas est petit, plus la polyligne est précise
  • Symbologie de l'élément source : si coché, l'élément 3D reprend le niveau, la couleur et le style de l'élément 2D source. Sinon, les attributs actifs de MicroStation sont utilisés

Recherche

  • Tolérance clic (u.m.) : rayon de recherche autour du clic pour trouver un texte/tag/cellule d'altitude ou un élément linéaire. Augmenter si rien n'est trouvé, réduire si le mauvais élément est attrapé

État (lecture seule)

  • Ligne 1 : élément sélectionné (type, longueur) ou position fixée
  • Ligne 2 : altitudes départ / arrivée, pente courante

4. Utilisation — Convertir (2D → 3D)

La commande Convertir crée l'équivalent 3D d'un élément 2D avec des altitudes interpolées linéairement sur l'abscisse curviligne (longueur développée, pas la corde).

1
Sélection
élément 2D
2
Clic
Z départ
3
Clic
Z fin
4
Intermédiaires
Data=ajouter
Reset=créer
  1. Lancement : vba run [Trans3D]Convertir — le formulaire Trans3D s'ouvre. L'élément 2D le plus proche est surligné dynamiquement sous le curseur.
  2. Sélection de l'élément 2D : cliquez près d'une ligne, polyligne, arc, courbe, B-spline, cercle ou ellipse dans une référence 2D (ou le modèle actif). Le type et la longueur s'affichent dans le cadre État. Reset = quitter la commande.
  3. Clic Z départ : cliquez à proximité d'un texte, tag ou cellule d'altitude. L'altitude est lue automatiquement et affichée dans le formulaire. Reset = revenir à la sélection (étape 1).
    Cercle/ellipse complet : une seule altitude suffit (Z constant), la création suit immédiatement.
  4. Clic Z fin : idem. L'aperçu 3D et la pente s'affichent en dynamique. Reset = revenir au Z départ (étape 2).
  5. Altitudes intermédiaires (0..N) : permet d'ajouter des points de changement de pente (regards, cassures).
    • Data = cliquer sur une cote d'altitude intermédiaire (projetée sur l'élément). Répétable
    • Reset = créer l'élément 3D et retourner à la sélection (étape 1)
✅ L'interpolation est linéaire par tronçons sur l'abscisse curviligne : s'il y a des altitudes intermédiaires, le Z varie linéairement entre chaque paire de cotes successives (départ → interm. 1 → interm. 2 → … → fin).
✅ Si Symbologie de l'élément source est coché (cadre Convertir), l'élément 3D reprend le niveau, la couleur et le style de l'élément 2D. Le niveau est créé automatiquement dans le fichier actif s'il n'existe pas.
✅ Si Découpage est activé, des points cotés (cercle + texte) sont placés le long de l'élément 3D créé, aux extrémités et aux positions intermédiaires, plus une flèche de direction.
⚠️ Les courbes avec pente (arc, B-spline) sont discrétisées en polyligne 3D au pas défini dans le formulaire. Un pas trop grand donne une polyligne anguleuse ; un pas trop petit alourdit le fichier.

5. Utilisation — Semer (points cotés sur un élément 3D)

La commande Semer matérialise des points cotés le long d'un élément linéaire 3D existant. Le Z de chaque point est interpolé sur la géométrie 3D réelle de l'élément.

1
Sélection
élément 3D
2
Aperçu
Data=créer
Reset=annuler
  1. Lancement : vba run [Trans3D]Semer — le formulaire Trans3D s'ouvre.
  2. Sélection de l'élément 3D : cliquez près d'un élément linéaire du modèle actif (issu de Trans3D ou non). L'élément est surligné pendant l'aperçu. Reset = quitter la commande.
  3. Aperçu et création : l'aperçu dynamique montre les points semés selon les paramètres du cadre Découpage du formulaire (modifiables en direct).
    • Data = créer tous les points cotés, puis retour à la sélection (étape 1) pour un autre élément
    • Reset = abandonner cet élément, retour à la sélection (étape 1)

Modes de répartition

ModeComportement
Distance fixe Un point au départ, puis tous les N mètres (ex. 0,50 m), et un point à la fin même si le dernier intervalle est plus court
Nombre de points N points également répartis sur la longueur développée

Objets créés par point

  • Cercle repère au Z réel (le point exploitable pour la modélisation)
  • Texte d'altitude à Z=0.00 (lisible en vue top) — chacun activable dans le formulaire
✅ Les paramètres du cadre Découpage (distance fixe / nombre de points) sont ajustables en direct pendant l'aperçu : l'affichage se met à jour immédiatement.

6. Utilisation — Points (point 3D isolé)

La commande Points crée un point 3D à partir d'une position 2D (snap précis : sommet, intersection…) et d'une altitude cliquée sur le plan.

1
Snap
position XY
2
Clic
altitude
  1. Lancement : vba run [Trans3D]Points — le formulaire Trans3D s'ouvre.
  2. Position XY : cliquez/snappez sur le point terrain exact (sommet, intersection, centre…). La position est mémorisée et affichée dans le cadre État. Reset = quitter la commande.
  3. Altitude : cliquez à proximité d'un texte/tag/cellule d'altitude. Le point coté (cercle au Z réel + texte altitude) est créé immédiatement, puis retour à l'étape 1 pour le point suivant. Reset = revenir à la position (étape 1).
✅ La commande est répétitive : après la création d'un point, on repart directement au snap de la position suivante. Pratique pour semer une série de points isolés (regards, bouches à clé…).
⚠️ Le Z du point snappé (qui dépend de l'élément visé par le snap) n'est jamais utilisé : seule la cote cliquée fournit l'altitude.

7. Lancement par key-in

  1. Ouvrez la fenêtre de saisie : Utilitaires > Saisie au clavier (Key-in)
  2. Tapez l'un des key-in suivants :
    CommandeKey-in
    Convertirvba run [Trans3D]Convertir
    Semervba run [Trans3D]Semer
    Pointsvba run [Trans3D]Points
  3. Validez par Entrée
✅ La fenêtre Key-in garde l'historique : ↑ puis Entrée pour relancer la dernière commande.

8. Lancement par touche de fonction

  1. Menu Utilitaires > Touches de fonction
  2. Sélectionnez trois touches libres, par exemple F6, F7 et F8 (les combinaisons Ctrl/Alt/Maj+F sont aussi possibles)
  3. Dans le champ Action, saisissez pour chaque touche :
    ToucheKey-in
    F6vba run [Trans3D]Convertir
    F7vba run [Trans3D]Semer
    F8vba run [Trans3D]Points
  4. Cliquez OK — le menu de touches de fonction est enregistré (fichier .mnu du workspace)

Un appui sur F6 lance Convertir, F7 lance Semer, F8 lance Points.

9. Dépannage rapide

SymptômeCause / remède
« Ouvrez d'abord un fichier DGN » Aucun dessin actif : ouvrez un DGN avant de lancer la commande.
« Aucun texte altitude à proximité » Cliquez plus près de l'origine du texte/tag, ou l'élément n'est pas numérique. Vérifiez aussi que le niveau n'est pas gelé.
« Aucun élément linéaire à proximité » Cliquez plus près de l'élément, ou augmentez la tolérance dans le formulaire. Seuls les types linéaires sont acceptés (pas les surfaces, cellules, textes).
Pas d'aperçu dynamique Vérifiez que le projet est bien compilé (Débogage > Compiler). Supprimer puis réimporter la classe de placement si nécessaire.
Erreurs de compilation « membre introuvable » Vérifiez Outils > Références : la « Bentley MicroStation DGN 8.9 Object Library » doit être cochée et pointer vers l'installation V8i active.
Les .cls s'importent comme modules standard Fins de ligne Unix (LF) dans les fichiers : convertir en CRLF puis réimporter.
Import du .frm échoue Le fichier frmTrans3D.frx doit être dans le même dossier que frmTrans3D.frm. En dernier recours : Insertion > UserForm, nommer le formulaire frmTrans3D, puis coller le code à partir de Option Explicit — les contrôles sont créés au runtime.
Le texte créé n'a pas le bon style Vérifiez que Dupliquer le style de la cote est coché dans le formulaire, et que le texte source est bien un texte simple (pas un champ de cellule sans style accessible).
L'élément 3D a un Z uniforme alors qu'il devrait avoir une pente Vérifiez que les altitudes de départ et de fin sont différentes. Pour un cercle/ellipse, le Z est toujours constant (une seule altitude).
Niveau manquant dans le fichier 3D Le niveau de l'élément source est créé automatiquement dans le fichier actif. Si le niveau du texte ou du cercle manque, sélectionnez-le dans la combo du formulaire.
